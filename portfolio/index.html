Universidade Lusófona de Humanidades E Tecnologia

O presente trabalho é realizado no contexto académico de Trabalho de fim do curso(TFC)
Projeto Em Curso
Plataforma de Gestão de Requisitos

⚫

Trata-se de uma aplicação web que tem como principal objetivo a avaliação e análise de requisitos. A avaliação é feita com base em válores númericos, que variam entre 1 a 5 onde 1 é o grau de menor importância e 5 o grau de maior relevância para o projeto em questão.

⚫

O projeto visa resolver o problema de pequenas empresas que têm como um dos focos de projeto, fazer a gestão de requisitos, uma vez que, muitos projetos acabam por falhar justamente por causa de requisitos mau defenidos.

⚫

O projeto foi desenvolvido em grupo de dois, na qual, tive a responsabilidade de lidar com 80% da componente prática (camada de apresentação e camada de dados). criação de um modelo de base de dados na qual pudesse conter a informação relativa a utilizadores e requisitos, tal base de dados que devia ser relacional. criação de um servidor local, criação de diferentes perfis com diferentes tipos de acesso.

⚫

Tecnologias e Metodologias usadas foram: HTML, SQL server, JavaScript, CSS, VSCode, IOdraw, UML, Git.

⚫
A realização deste trabalho fez com que iniciássemos um estudo mais aprofundado sobre a temática da gestão de requisitos,os processos que esta engloba, bem como a sua importância para o sucesso de um projeto. Através do estudo realizado foi produzida uma ferramenta que não contendo todas as funcionalidades quando comparada às ferramentas concorrentes a nível comercial, se distingue pelo seguinte:

• Oferece aos seus utilizadores uma experiência de simples navegação e fácil aprendizagem.

• Disponibiliza um sistema de aprovações de requisitos que permite aprovar cada requisito em particular.

• Apresenta um novo meio de comunicação que permite a discussão dos requisitos, permitindo melhorar a qualidade dos mesmos.

• Preocupa-se com a gestão da mudança no contexto dos requisitos, oferecendo um método de aprovação dos requisitos aprovados pelo sistema.

• Suporta a análise de impacto de requisitos através de uma matriz de factores críticos de sucesso(FCS) calculada pela ponderação das pontuações dada por casa interveniente.
